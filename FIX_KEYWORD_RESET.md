# 🔧 Исправление проблемы сброса ключевых слов

## Проблема
Сервис падает при попытке сменить тему и переустановить ключевые слова. Это происходит из-за невалидного синтаксиса удаления большого количества записей.

## Решение
Код был исправлен на использование **деактивации** вместо **удаления** ключевых слов.

### До (НЕПРАВИЛЬНО ❌)
```python
# Удаление всех ключевых слов разом - может вызвать блокировку БД
db.session.query(SearchKeyword).delete(synchronize_session=False)
db.session.commit()  # Слишком долгая операция
```

### После (ПРАВИЛЬНО ✅)
```python
# Отключение активных ключевых слов - быстро и безопасно
db.session.query(SearchKeyword).filter_by(active=True).update(
    {SearchKeyword.active: False},
    synchronize_session='fetch'
)
# Новые ключевые слова добавляются с active=True
for keyword in keywords_list:
    kw = SearchKeyword(
        keyword=keyword,
        active=True,  # Активны по умолчанию
    )
    db.session.add(kw)

db.session.commit()  # Одна быстрая операция
```

## Преимущества этого подхода

| Аспект | Старый метод | Новый метод |
|--------|--------------|------------|
| **Скорость** | ❌ Медленно (удаление) | ✅ Быстро (обновление) |
| **Безопасность** | ❌ Может зависнуть | ✅ Надежно |
| **История** | ❌ Теряются данные | ✅ Сохраняется история |
| **Откат** | ❌ Сложный | ✅ Простой (reactivate) |
| **Блокировки** | ❌ Возможны | ✅ Минимальны |

## Как это работает

```
Старые ключевые слова:  [привет, мир, тест, ...]
                        ↓
                  Mark as inactive
                        ↓
Старые (неактивные):   [привет❌, мир❌, тест❌, ...]
Новые (активные):      [hello✅, world✅, test✅, ...]

Discovery Service видит только АКТИВНЫЕ (новые) ключевые слова

Старые сохраняются в базе для истории и отката
```


Должны быть новые ключевые слова.

###Посмотреть неактивные ключевые слова:
```sql

```
Должны быть старые ключевые слова.

## Если что-то пошло не так

### Восстановить старые ключевые слова:
```sql
UPDATE search_keywords 
SET active=TRUE 
WHERE active=FALSE 
LIMIT 11526;  -- или нужное количество
```

### Отключить новые ключевые слова:
```sql
UPDATE search_keywords 
SET active=FALSE 
WHERE created_at > '2026-02-22';  -- дата вашего сброса
```

## Дополнительно

- Старый синтаксис с `.limit()` + `.delete()` вызывал ошибку SQLAlchemy

- Все изменения обернуты в try/except с полным откатом при ошибке
